Setting Up Your Development Environment

Prerequisites

Before we begin building our React application with ArcGIS maps, we need to make sure we have the right tools installed. This section will guide you through setting up your development environment.

Required Tools

  • Node.js (v18.0.0 or later) - JavaScript runtime environment
  • npm (comes with Node.js) - Package manager for JavaScript
  • Code Editor - We recommend Visual Studio Code
  • Modern Web Browser - Chrome, Firefox, Edge, or Safari

Installing Node.js and npm

Node.js is a JavaScript runtime that allows us to run JavaScript on the server. npm (Node Package Manager) comes bundled with Node.js and helps us manage our project dependencies.

Installation Steps:

  1. Visit the official Node.js website

  2. Download the LTS (Long Term Support) version, which is recommended for most users

  3. Follow the installation instructions for your operating system

  4. Verify the installation by opening your terminal or command prompt and running:

    node --version

    and

    npm --version

If both commands display version numbers, you've successfully installed Node.js and npm!

Creating a New React Project with Vite

Vite is a modern build tool that provides an extremely fast development experience. We'll use it to create our React 19 application.

Step 1: Create a New Project

Open your terminal or command prompt and run the following command:

npm create vite@latest react-arcgis-map -- --template react-ts

Let's break down this command in detail so you understand exactly what's happening:

Command Breakdown:

  • npm create - This is npm's command for creating a new project using a template. It's a shorthand for npm init that's specifically designed to work with project scaffolding tools.
  • vite@latest - This specifies that we want to use Vite as our project scaffolding tool, and specifically the latest version. Vite is a modern build tool that provides faster development experience than older tools like Create React App.
  • react-arcgis-map - This is the name we're giving to our project. It will also be the name of the directory that gets created.
  • -- - This double dash is a common convention in command-line tools that signifies "what comes after this are arguments to pass to the script being executed, not to npm itself." In this case, it means the following arguments will be passed to the Vite project creation tool.
  • --template react-ts - This tells Vite which template to use for our project:
    • react - Means we want a React project (as opposed to Vue, Svelte, etc.)
    • -ts - This suffix indicates we want TypeScript support integrated into our React project
    TypeScript is a superset of JavaScript that adds static type definitions, helping catch errors early during development.

After running this command, you'll see prompts in your terminal guiding you through the rest of the setup process. Vite will:

  • Create a new directory called react-arcgis-map
  • Generate all the necessary files for a React + TypeScript project
  • Set up the project structure with modern best practices
  • Create a basic React application that's ready to run

Step 2: Navigate to the Project Directory

cd react-arcgis-map

Step 3: Install Dependencies

npm install

This command installs all the necessary dependencies for a React project as specified in the package.json file.

Step 4: Start the Development Server

npm run dev

This command starts the development server. You should see output indicating that the server is running, including a local URL (typically http://localhost:5173/) where you can view your application.

Open your browser and navigate to the provided URL. You should see a welcome page for your new React application!

What Just Happened?

You've created a new React 19 application using Vite with TypeScript support. Vite provides a fast development environment with features like:

  • Instant server start
  • Lightning-fast hot module replacement (HMR)
  • True on-demand compilation
  • TypeScript support out of the box

Installing ArcGIS API for JavaScript

Next, we need to install the ArcGIS API for JavaScript to integrate mapping capabilities into our React application.

Understanding the ArcGIS JavaScript SDK

Before we install, let's understand what we're adding to our project:

What is the ArcGIS JavaScript SDK?

The ArcGIS API for JavaScript is Esri's library for building web mapping applications. It provides:

  • Interactive map components
  • Geospatial data visualization tools
  • Geocoding and routing services
  • Spatial analysis capabilities
  • Access to Esri's basemaps and services

The @arcgis/core package is the ES modules version of this API, which works well with modern JavaScript frameworks like React.

Installing the Package

In your terminal, with your project directory as the current directory, run:

npm install @arcgis/core

Let's break down what this command does:

  • npm install - This tells npm to download and install a package from the npm registry
  • @arcgis/core - This is the package name. The @arcgis/ part is called a "scope" and indicates that this package is published by the Esri/ArcGIS organization. The core part indicates that this is the core library of the ArcGIS JavaScript API.

When you run this command, npm will:

  • Download the latest version of the @arcgis/core package
  • Install it in the node_modules/@arcgis/core/ directory of your project
  • Add it to your project's dependencies in package.json
  • Download and install any other packages that @arcgis/core depends on

Important Note on Size

The ArcGIS API is quite large (over 10MB) because it includes many specialized mapping and analysis capabilities. When we build our application, we'll use techniques to ensure we only include the parts we actually need.

Installing Calcite Components (Optional)

Understanding Calcite Design System

Before installing, let's understand what Calcite is and why it's useful for our project:

What is the Calcite Design System?

Calcite is Esri's design system - a collection of UI components, design guidelines, and patterns that:

  • Provides a consistent visual style matching other Esri applications
  • Offers pre-built accessible components that follow web standards
  • Includes specialized components for mapping applications
  • Saves development time by providing ready-to-use UI elements
  • Ensures your application follows modern accessibility guidelines

While using Calcite is optional (you could use any UI framework like Material UI, Tailwind, Bootstrap, etc.), it's beneficial when building ArcGIS applications because:

  • It's designed specifically with mapping applications in mind
  • It integrates seamlessly with the ArcGIS JavaScript API
  • Users familiar with other Esri products will find your interface intuitive

Installing the Package

In your terminal, with your project directory as the current directory, run:

npm install @esri/calcite-components

Let's break down this command:

  • npm install - Tells npm to download and add a package to our project
  • @esri/calcite-components - The package name, where:
    • @esri/ - The scope, indicating this package is published by Esri
    • calcite-components - The actual package name for the UI component library

Web Components Under the Hood

Unlike React-specific libraries, Calcite is built using Web Components - a set of standardized web platform APIs that allow you to create custom, reusable HTML elements. This means:

  • They work across different frameworks (not just React)
  • They're more future-proof as they use web standards
  • They require a slightly different integration approach than typical React components

In a later section, we'll cover exactly how to properly integrate these components with React.

Basic Project Structure

Now that we have our project set up, let's understand the basic structure that Vite has created for us:

react-arcgis-map/ ├── node_modules/ # Project dependencies ├── public/ # Static assets served as-is ├── src/ # Source code │ ├── assets/ # Images, fonts, etc. │ ├── App.css # Styles for App component │ ├── App.tsx # Main App component │ ├── index.css # Global styles │ ├── main.tsx # Entry point │ └── vite-env.d.ts # TypeScript declarations for Vite ├── .gitignore # Git ignore file ├── index.html # HTML template ├── package.json # Project metadata and dependencies ├── tsconfig.json # TypeScript configuration └── vite.config.ts # Vite configuration

We'll primarily be working in the src directory, where we'll add our components, styles, and map integration code.

Cleaning Up the Default Project Structure Optional

▼

After generating our React project, it's a good idea to clean up some of the default files and content that we won't need for our mapping application. Simplifying the starter code gives us a cleaner foundation to build upon.

Files to Remove or Modify

  • public/vite.svg - We'll replace this with our own favicon later
  • src/assets/react.svg - We don't need the example React logo
  • src/assets/ - This folder can be removed if empty after removing react.svg

Simplify These Files

  • index.html - Clean up references to Vite and unnecessary metadata
  • src/App.tsx - Remove the counter example, logos, and unused imports
  • src/App.css - Replace with a minimal styling foundation
  • src/index.css - Clean up unnecessary default styles
  • src/main.tsx - Remove any unnecessary imports or configurations

Benefits of Cleaning Up

  • Simplicity - Makes it easier to understand the core structure
  • Focus - Removes distractions like the counter example
